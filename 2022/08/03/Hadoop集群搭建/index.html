Hadoop 集群搭建

发表于 2022-08-03 更新于 2023-05-24 分类于 大数据 阅读次数：
本文字数： 2.5k 阅读时长 ≈ 2 分钟

[Updated] 本文记录了使用 VMware Workstation 创建 3 台 Linux 服务器搭建 Hadoop 集群的过程。

Hadoop 集群搭建

环境与基本配置

  • 虚拟软件：VMware Workstation
  • 虚拟系统：CentoOS 7.5
  • java 版本：JDK 1.8
  • Hadoop 版本：Hadoop 3.1.3

为什么 Hadoop 推荐至少 3 个节点？

保证集群的高可用性和容错性。

Hadoop 集群中由一个 NameNode 节点负责管理元数据，和多个 DataNode 节点负责存储数据。

  • 如果只有一个 DataNode 节点，那么一旦它出现故障，就会导致数据丢失。
  • 如果只有两个 DataNode 节点，那么一旦其中一个出现故障，就会导致集群无法正常工作，因为无法达成多数投票。

因此，Hadoop 集群至少需要 3 个节点，才能保证在一个节点出现故障的情况下，集群仍然能够正常运行。

集群部署规划

  • NameNode 和 SecondaryNameNode 不安装在同一台服务器上
  • ResourceManager 很消耗内存，不和 NameNode、SecondaryNameNode 配置在同一台机器上

安装配置

初步规划

  1. 创建 3 台虚拟机
  2. 先安装配置好 1 台，再分发、同步到其他虚拟机
  3. 集群启动测试

1. 虚拟机创建

注：本文使用 VMWare Workstation 虚拟机。

  1. 创建一台模板虚拟机

    1. 虚拟机设置

      设备参数：

      • 系统：CentOS 7.5
      • 内存：设定后不可更改。视物理机内存使用情况和与其数据大小而定，越大越好，但要考虑 3 台的量物理机是否带的动。
      • 硬盘：设定后可拓展。参考 = 系统大小 + 软件大小 + 预期数据大小 + 预留空间大小。
      • 网络适配器：NAT/Bridge 都行，保证集群之间能互相联通即可。

    2. 虚拟 CentOS 设置

      1. 创建 hadoop 用户，并配置 root 权限，方便 sudo 执行命令
      2. 卸载自带 JDK

  2. 虚拟机克隆

    两种方案：

    • 将模板虚拟机克隆 3 份，克隆的 3 份组成集群，模板机作为备份，方便出错重做
    • 将模板虚拟机克隆 2 份，模板机与克隆的 2 份组成集群

  3. 配置集群的静态 IP 地址和主机名

    修改 3 台虚拟机的 hostname 和 hosts 文件，确认能够互相 ping 通。

2. 搭建 Hadoop 集群

  1. 单机安装 Hadoop

    选择一台虚拟主机作为 master 节点，其他作为 worker 节点。先在 master 作以下操作：

    1. 下载安装 JDK，并配置环境变量
    2. 下载并解压 Hadoop 压缩包，并配置环境变量
    3. 单机测试能否运行

  2. 集群配置

    1. 在 master 节点上配置 ssh 免密登录，使得 master 节点可以无密码访问其他 worker 节点

      1. master 生成公钥和私钥：ssh-keygen -t rsa
      2. 将公钥 id_rsa.pub 拷贝到要免密登录的节点上：ssh-copy-id <目标免密登录节点>

    2. 将 master 上安装好的 JDK 和 Hadoop 用 scp 拷贝到其他 worker 的相同目录下，保持各节点目录结构一致

    3. 修改集群各机器的配置文件

      1. 修改 master 上的配置文件

        1. core-site.xml
          • 指定 NameNode 的地址（8020）
          • 指定 hdfs 网页登录使用的静态用户
        2. hdfs-site.xml
          • 指定 Namenode web 端访问地址（9870）
          • 指定 Secondary Namenode web 端访问地址（9868）
        3. yarn-site.xml
          • 指定 ResourceManager web 端的地址
        4. mapred-site.xml
          • 指定 MapReduce 程序运行在 Yarn 上
          • 指定历史服务器 historyserver 的 web 端地址（19888）

      2. 使用 rsync 命令将以上配置文件同步到集群的其他 worker

    4. 配置 workers

      1. 在 master 的 {HADOOP_HOME}/etc/hadoop/workers 文件中添加集群所有节点（包括 master 与 workers）的主机名（文件中不允许空行，只许换行）
      2. 将修改后的文件同步至所有节点

集群启停

启动集群

  • 注：第一次启动需要在 master 节点格式化 NameNode：hdfs namenode -format

  1. 在 master 启动 HDFS：sbin/start-dfs.sh

    测试：浏览器打开 http://<master 主机名>:9870 可以查看 hdfs 文件即成功。

  2. 在配置 ResourceManager 的节点启动 YARN：sbin/start-yarn.sh

    测试：浏览器打开 http://<ResourceManager 节点主机名>:8088 可以查看 hdfs 文件即成功。

  3. 在 master 启动 historyserver：mapred --daemon start historyserver

    测试：浏览器打开 http://<master 主机名>:19888/jobhistory 可以查看历史任务日志即成功。

  4. 通过 jps 命令可以查看各节点的进程是否正常运行

关闭 YARN
